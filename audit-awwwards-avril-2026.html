01 — Loading Experience

Le premier
contact.

Sur Awwwards, les 2 premières secondes décident de tout. Le preloader est le rideau qui se lève. Chez les meilleurs, c'est un moment de tension narrative. Chez vous, c'est un logo qui pulse puis disparaît. Fonctionnel, mais pas mémorable.

Manque critique

Pas de séquence de loading narrative

Votre preloader actuel (#preloader) fait un simple fade-out après 1.2s. C'est invisible. Les SOTD utilisent le preloader comme un premier acte narratif : compteur de pourcentage avec typographie cinétique, reveal progressif d'un élément clé (un fruit qui se dévoile ?), ou une animation de la marque qui installe l'ambiance.

Imaginez : le logo octopus se dessine trait par trait en or, puis les lettres "Atelier Mirage" apparaissent en split-text, puis le hero se révèle avec un clip-path circulaire qui s'expand depuis le centre. Boom — le visiteur est déjà dans votre monde.

Réf. : #preloader (styles.css L.706-709) · Actuellement : img pulse + line scaleX
Élévation

Le hero appear manque de dramaturgie

Vous avez un beau séquençage GSAP (hero-eyebrow → hero-h1 → hero-hook → hero-desc → hero-actions), mais c'est du translateY+opacity classique. Les SOTD font du split-text lettre par lettre sur le H1, avec un overshoot spring sur chaque caractère. Votre H1 "L'Art du Dessert." mérite ce traitement — chaque lettre qui rebondit légèrement en arrivant, c'est 10× plus mémorable.

// Avec GSAP SplitText (plugin Club):
const split = new SplitText('.hero-h1', { type: 'chars' });
gsap.from(split.chars, { y: 80, opacity: 0, rotateX: -90,
  stagger: 0.03, duration: 1.2, ease: 'back.out(1.7)' });
Réf. : index.html L.835 · Timeline GSAP hero
Idée créative

Page transition : le chaînon manquant

Quand un visiteur clique sur une page produit, c'est un hard reload classique. Zéro transition. Sur Awwwards, chaque navigation est une animation. Un overlay qui glisse, un clip-path qui contracte, un fade croisé avec GSAP Flip — ça transforme un site multi-pages en une expérience fluide comme une SPA.

Sans passer à un framework JS, vous pouvez utiliser l'API View Transitions (supportée Chrome/Edge) ou un wrapper léger type Barba.js pour orchestrer des transitions entre vos pages HTML.

Impact Awwwards : ●●●●● (critère créativité, le plus faible actuellement)
02 — Motion Design

Le mouvement
comme langage.

Vous avez GSAP + ScrollTrigger. Vous avez 5 courbes d'easing custom. Le fondement est là. Mais vous n'exploitez que 10% du potentiel. Voici ce qui manque pour passer de "site animé" à "expérience chorégraphiée".

Élévation majeure

Le scroll est passif — il devrait être un instrument

Actuellement, vos animations scroll sont binaires : l'élément entre dans le viewport → il fait un fadeIn+translateY. C'est du 2019. Les SOTD 2025-2026 utilisent le scroll comme une timeline continue :

• Parallax différentiel : votre section .parallax-bg utilise un background-attachment: scroll classique. Les award-winners décalent chaque couche à des vitesses différentes via ScrollTrigger scrub. Imaginez la section "Mes Engagements" où le texte monte plus vite que l'image de fond, créant un effet de profondeur cinématographique.

• Scale + blur sur scroll : votre vidéo "Le Moment Mirage" devrait grandir de scale(0.85) à scale(1) au fur et à mesure que l'utilisateur scrolle vers elle, avec un léger blur qui s'estompe. L'effet "focus pull" des caméras de cinéma.

• Horizontal scroll section : votre collection de 10 produits en grille 3×3 est correcte sur desktop. Mais imaginez une section où les cartes défilent horizontalement au scroll vertical, comme un rail de galerie. C'est le pattern le plus récompensé sur Awwwards en 2025.

Réf. : ScrollTrigger.create() (index.html L.821-838) · Actuellement : start/onEnter binaire uniquement
Manque critique

Aucune animation de texte avancée

Vos titres de section (section-h2) apparaissent en bloc avec un simple fadeIn. Les sites primés animent le texte ligne par ligne, mot par mot, ou caractère par caractère. Votre typographie Cormorant Garamond à 300 weight est magnifique — mais elle mérite un reveal cinétique.

Concrètement : chaque section-h2 devrait avoir ses lignes qui glissent depuis le bas avec un masque (overflow:hidden sur un wrapper + translateY sur le contenu), décalées de 0.1s. C'est le pattern "line reveal" et c'est devenu un standard sur les SOTD.

/* Masque de reveal */
.line-wrap { overflow: hidden; }
.line-inner { transform: translateY(105%); }
/* GSAP */
gsap.to('.line-inner', { y: 0, duration: 1, ease: 'power3.out',
  stagger: 0.12, scrollTrigger: { trigger: el, start: 'top 80%' } });
Idée créative

Smooth scroll (Lenis) pour l'inertie luxe

Le scroll natif du navigateur est abrupt. Les sites luxury installent Lenis (ou Locomotive Scroll) pour un scroll avec inertie — un momentum doux qui prolonge le geste de l'utilisateur. C'est subtil, mais ça change tout le ressenti. C'est ce qui fait qu'on "sent" la qualité dès la première interaction.

Lenis est léger (~3KB gzip), compatible avec GSAP ScrollTrigger, et s'installe en 5 lignes de JS.

Impact sur le score créativité : +0.8 à +1.2 points estimé
03 — Typographic Craft

La typo comme
sculpture.

Votre choix Cormorant Garamond + Outfit + JetBrains Mono est excellent — c'est un triptyque cohérent. Mais les SOTD ne se contentent pas de choisir de bonnes polices : ils les mettent en scène.

Force

Le système typographique est solide

La hiérarchie serif (titres) / sans (corps) / mono (eyebrows, labels) est parfaitement exécutée. Le poids 300 sur Cormorant donne une élégance qui colle au luxe artisanal. Le clamp() sur les tailles est bien calibré. Les letter-spacing négatifs sur les H1 (-0.02em) ajoutent la tension visuelle que les jurés apprécient.

Score typographie isolée : 8.5/10 — parmi les meilleurs aspects du site
Élévation

Le H1 du hero mérite un traitement éditorial

Votre H1 "L'Art du Dessert." fait clamp(3.2rem, 5.5vw, 6rem). C'est bien, mais les SOTD poussent les titres hero à 20-30vw — des typographies massives qui occupent toute la largeur de l'écran. Imaginez "L'Art du" en Outfit light 8vw, et "Dessert." en Cormorant italic 18vw qui descend sous la fold. Ce contraste de scale crée un impact visuel immédiat.

Ça ne veut pas dire tout changer — mais le hero est votre vitrine. C'est là que la typographie doit être la plus audacieuse.

Élévation

Les eyebrows manquent de personnalité

Tous vos eyebrows utilisent le même pattern : JetBrains Mono 0.65rem, letter-spacing 0.35em, uppercase, avec une ligne horizontale avant. C'est propre, mais c'est répétitif sur 17 pages. Variez : certains pourraient avoir un numéro (01 —), d'autres un trait vertical, d'autres une icône SVG inline. La variation dans la constance, c'est ce qui rend un design system vivant.

04 — Narrative Architecture

Raconter,
pas lister.

Un site Awwwards ne présente pas des sections — il raconte une histoire. Chaque scroll devrait donner envie de voir la suite. C'est le point le plus transformateur pour vous.

Manque critique

La homepage est une liste de blocs, pas un voyage

Votre homepage suit une structure classique : Hero → KPI → Testimonials → Collection → Pricing → Video → Guarantees → Engagements → FAQ → CTA. C'est logique d'un point de vue CRO. Mais d'un point de vue narratif, c'est plat.

Un SOTD structurerait ça comme un récit en 3 actes :

Acte 1 — L'Illusion (Hero + Vidéo) : On montre l'effet "wow" immédiatement. La vidéo de découpe devrait être DANS le hero, pas 5 sections plus bas. Le visiteur doit comprendre le produit en 3 secondes.

Acte 2 — La Preuve (Chiffres + Témoignages + Collection) : Maintenant qu'il est accroché, on prouve. Mais au lieu de sections empilées, imaginez une transition fluide où les KPI se transforment en témoignages via un scroll horizontal.

Acte 3 — L'Invitation (Pricing + CTA) : Simple, direct, élégant. Un seul message : "Essayez gratuitement."

Réf. : Structure actuelle index.html — 11 sections principales empilées verticalement
Élévation

La vidéo "Moment Mirage" est votre atout #1 — utilisez-la mieux

Votre vidéo de découpe de la pêche est extraordinaire. C'est littéralement le moment "wow" de votre produit. Mais elle est enfouie en section 5 avec un simple player dans un cadre. Sur un SOTD, cette vidéo serait :

• Full-screen, pas dans un cadre 9/14. Elle devrait exploser l'écran.

• Contrôlée par le scroll : au lieu de tourner en boucle, chaque frame de la vidéo correspondrait à une position de scroll. L'utilisateur "découpe" le fruit en scrollant. C'est le pattern le plus viral de 2025 (Apple l'utilise sur chaque page produit).

• Le hashtag #MomentMirage et le tagline "L'illusion se goûte" sont excellents — mais ils devraient apparaître APRÈS la révélation, pas pendant.

Réf. : .video-reveal-section (index.html L.492-521) · .video-reveal-frame (homepage.css)
« Le site a tous les ingrédients — la vidéo, le produit, l'esthétique. Mais il les présente comme un catalogue. Un award-winner les présenterait comme un film. »
Idée créative

Une page produit = une expérience, pas une fiche

Vos 9 pages produit (produit-vanille.html, produit-fraise.html, etc.) suivent un layout classique : galerie sticky + info à droite. C'est fonctionnel. Mais imaginez : quand l'utilisateur arrive sur "La Pêche", il voit d'abord le fruit en plein écran, parfait, photoréaliste. En scrollant, le fruit se coupe, révélant les couches intérieures (confit, mousse, biscuit), chacune annotée. C'est du scroll-driven product storytelling.

Ça demande du travail, mais même une seule page produit "showcase" traitée ainsi pourrait être soumise seule à Awwwards.

05 — Micro-interactions

Les détails qui
font la différence.

Les micro-interactions sont le sel d'un SOTD. Elles ne servent pas la fonctionnalité — elles servent l'émotion. Voici ce qui manque et ce qui existe déjà.

Manque critique

Pas de curseur custom

C'est devenu un standard non-négociable sur les sites luxury primés. Un curseur custom (cercle doré qui suit la souris avec un léger retard, qui grossit au survol des éléments cliquables, qui se transforme en "View" sur les cartes produit et en "Play" sur la vidéo) ajoute une couche de polish qui impressionne systématiquement les jurés.

Techniquement : un div fixe en position absolute, animé via requestAnimationFrame avec un lerp de 0.1 pour le retard. 30 lignes de JS.

Force

Les hover states sont soignés

Le btn-primary avec son ::before qui slide en gold, le shop-card qui fait un translateY(-8px) avec une ombre qui grandit, les FAQ icons qui rotate(45deg) avec un background gold — tout ça est bien exécuté. Les transitions utilisent vos courbes custom (--ease-spring, --ease-out) de manière cohérente. C'est un vrai point fort.

Score micro-interactions hover : 8/10
Élévation

Les cartes produit méritent un effet magnétique

Vos shop-cards font un translateY(-8px) au hover. C'est propre, mais c'est du 2020. Les SOTD utilisent un tilt 3D magnétique : la carte s'incline légèrement dans la direction du curseur (via JS qui calcule la position relative de la souris), avec un reflet subtil qui se déplace. Vous avez déjà le perspective:1200px sur .shop-grid — l'infrastructure est là, il manque le JS.

card.addEventListener('mousemove', e => {
  const rect = card.getBoundingClientRect();
  const x = (e.clientX - rect.left) / rect.width - 0.5;
  const y = (e.clientY - rect.top) / rect.height - 0.5;
  card.style.transform = `perspective(1000px) rotateY(${x*8}deg) rotateX(${-y*8}deg) translateY(-8px)`;
});
Idée créative

Magnetic buttons sur les CTA principaux

Quand le curseur approche d'un bouton "Devenir partenaire", le bouton devrait se déplacer légèrement vers le curseur, comme un aimant. C'est un effet subtil (3-5px max) qui rend l'interface vivante. Monzo, Stripe, et la majorité des SOTD l'utilisent.

06 — Visual Craft & Texture

Le grain,
la matière.

Un SOTD se reconnaît aux détails qu'on ne voit pas consciemment mais qu'on ressent : la texture, les ombres, les dégradés, le rythme vertical.

Force

L'overlay grain est un excellent choix

Votre grain SVG feTurbulence en overlay fixe (body::after) est exactement ce que font les meilleurs studios. L'opacité réduite sur mobile (hover:none) montre une attention à la performance. Le backdrop-filter blur(24px) saturate(1.6) sur la navbar est du même calibre que ce qu'on voit chez les studios comme Locomotive ou Resn.

Force

La palette est irréprochable

Cream/Ink/Gold avec les déclinaisons graduées (cream-2, cream-3, ink-soft, ink-muted, gold-light, gold-pale) — c'est un système chromatique digne d'un brand book professionnel. Les gradients (gradient-gold, gradient-glass, gradient-glow) ajoutent de la profondeur sans jamais casser l'harmonie. Le shadow system en 6 niveaux (xs→xl + shadow-gold) est particulièrement bien calibré.

Élévation

Les séparateurs de section sont trop timides

Chaque section utilise le même pattern ::before avec un gradient transparent → rgba(26,23,20,0.08) → transparent. C'est propre mais invisible. Les SOTD utilisent des éléments de transition entre les sections : une forme SVG organique, une ligne ondulée, ou même un changement de couleur de fond progressif (cream → cream-2 → ink). Le site devrait "respirer" visuellement entre chaque section, pas juste empiler.

Élévation

Pas d'éléments organiques animés

Votre produit est artisanal, organique, vivant. Mais le design est géométrique : rectangles arrondis, grilles, lignes droites. Ajoutez des formes blob SVG animées en arrière-plan — des formes organiques dorées qui pulsent lentement, rappelant la matière vivante de vos créations. Même un seul blob animé derrière la section hero ajouterait de la vie.

Idée créative

Le menu overlay est magnifique — exploitez-le plus

Votre choix du burger-only avec le full-screen overlay est un parti pris fort et assumé qui fonctionne parfaitement pour une marque luxury. L'animation staggerée des liens (0.05s→0.3s) et le fond rgba(26,23,20,0.97) avec backdrop-filter:blur(20px) sont excellents. Pour pousser plus loin : ajoutez une image de fond qui change au hover de chaque lien (quand on survole "Catalogue", une photo de la collection apparaît à droite ; quand on survole "L'Atelier", une photo de l'atelier).

Réf. : #mobile-overlay (styles.css L.734-778) · Stagger animation : 6 paliers

Le plan
vers le SOTD.

Priorisé par impact Awwwards / effort de développement. Chaque phase vous rapproche du seuil des 8.0.

Phase 1 — Quick wins (1-2 jours)
Lenis smooth scroll — 5 lignes de JS, impact immédiat sur le ressenti
Custom cursor — 30 lignes JS + 10 lignes CSS, impression "premium" instantanée

Magnetic buttons — Sur les 3 CTA principaux, 15 lignes JS

Preloader narratif — Logo draw SVG + split-text "Atelier Mirage" + clip-path reveal

ScrollTrigger scrub — Parallax continu sur engagements, vidéo scale+blur

Phase 3 — Transformation narrative (1-2 semaines)
Page transitions — Barba.js ou View Transitions API entre toutes les pages
Vidéo scroll-driven — Le "Moment Mirage" contrôlé frame par frame au scroll
Restructuration hero — Vidéo en hero, typo oversized, 3 actes narratifs
Horizontal scroll collection — Rail de produits au scroll vertical desktop
Phase 4 — Polish award (2-3 jours)
Sound design — Micro-sons Howler.js sur les interactions clés (toggle, CTA, reveal)
Page produit showcase — 1 page "La Pêche" avec scroll-driven layer reveal
Blob SVG animé — Forme organique dorée en arrière-plan du hero
Performance audit — Lighthouse 90+ pour ne pas perdre de points techniques
Conclusion

Le verdict
final.

« Atelier Mirage a la direction artistique d'un site à 8/10. Mais sans mouvement avancé, sans curseur custom, sans transitions de page, le jury voit un beau template — pas une expérience. La Phase 1 seule suffirait à passer de 6.8 à 7.5. Les Phases 2-3 ouvrent la porte du SOTD. »

Ce qui est remarquable, c'est que la fondation est déjà là. Le design system, la palette, la typographie, le grain, le glassmorphism — tout ça est du niveau award. Ce qui manque, c'est la couche d'animation et d'interaction qui transforme un design statique en expérience vivante. La bonne nouvelle : avec GSAP déjà en place, 80% du chemin vers le SOTD est du JS, pas du redesign.
